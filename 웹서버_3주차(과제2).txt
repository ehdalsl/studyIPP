1. 데이터 저장 방식

char: 고정 길이 문자열입니다. 예를 들어, char(10)으로 설정하면 모든 데이터는 10바이트 공간을 차지합니다. 입력된 문자열이 10바이트보다 길면 잘리고, 짧으면 공백으로 채워집니다.
varchar2: 가변 길이 문자열입니다. 실제 입력된 문자열 길이만큼 공간을 차지합니다. 최대 길이를 설정할 수 있으며, 일반적으로 varchar2(255) 또는 varchar2(4000) 등으로 사용됩니다.
2. 성능

char: 데이터 길이가 고정되어 있어 검색 속도가 빠릅니다. 하지만 공간 활용도가 낮아 저장 공간 낭비가 발생할 수 있습니다.
varchar2: 데이터 길이가 가변적이라 공간 활용도가 높습니다. 하지만 검색 속도가 char보다 느릴 수 있습니다.
3. 활용

char: 다음과 같은 경우에 적합합니다.
데이터 길이가 항상 일정한 경우 (예: 주민등록번호, 성별)
검색 속도가 중요한 경우
varchar2: 다음과 같은 경우에 적합합니다.
데이터 길이가 변동될 수 있는 경우 (예: 이름, 주소)
공간 활용도가 중요한 경우
도서 정보 테이블에서 author와 bookname 필드의 데이터 유형 선택 시 고려해야 할 사항:

데이터 길이: 데이터 길이가 항상 일정한 경우 char를 사용하는 것이 유리합니다. 하지만 데이터 길이가 변동될 수 있는 경우 varchar2를 사용해야 합니다.
검색 속도: 검색 속도가 중요한 경우 char를 사용하는 것이 유리합니다. 하지만 공간 활용도가 중요한 경우 varchar2를 사용해야 합니다.
저장 공간: 저장 공간이 부족한 경우 varchar2를 사용하는 것이 유리합니다. 하지만 검색 속도가 느려질 수 있다는 점을 고려해야 합니다.
결론:

char와 varchar2는 각각 장단점이 있으며, 상황에 맞게 적절한 데이터 유형을 선택해야 합니다. 도서 정보 테이블에서 author와 bookname 필드의 데이터 유형을 선택할 때는 위의 내용을 참고하여 결정하시기 바랍니다.
